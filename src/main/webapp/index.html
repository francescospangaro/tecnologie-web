Introduction to servlet programming:

Servlet mapping:

Note that if you enable the default mapping "/" for servlet5 than you will have to invoke each URL directly using the address bar in the browser, because the default mapping will override /TIW-servletIntro/index.html (this page)

/foo/bar/index.html servlet 1 /foo/bar/index.do servlet 1 /baz servlet 2
/baz/index.html servlet 2 /products servlet 3 /products/index.html Error (Servlet5 if the "/" mapping is set)
/products/cars.do servlet 4 /index.do servlet 4 /catalog Error (Servlet5 if the "/" mapping is set)

Request information:

http://localhost:8080/TIW-servletIntro/RequestInfo

Access counter: shared thread unsafe version (?reset=true to reset to 0):

Restart Tomcat to load the servlet config parameter

http://localhost:8080/TIW-servletIntro/UnsafeCounter
http://localhost:8080/TIW-servletIntro/UnsafeCounter?reset=true

This does the same using WebMapping

http://localhost:8080/TIW-servletIntro/UnsafeCounterMapping
http://localhost:8080/TIW-servletIntro/UnsafeCounterMapping?reset=true

step 1: Check cookies http://localhost:8080/TIW-servletIntro/CookieInspector
step 2: Set foo = bar http://localhost:8080/TIW-servletIntro/CookieInspector?name=foo&value=bar
step 3: Check cookies http://localhost:8080/TIW-servletIntro/CookieInspector

Session use, one counter per client:

This example is better viewed with an external browser.

Step 1 open a bowser window and call and click
http://localhost:8080/TIW-servletIntro/SessionCounter

Step 2: open an incognito window and click on the same link again, then you have two distinct clients with separate sessions

Step 3: reload any number of times the pages to increment the counter.

Step 4: to reset the counting, in one of the two windows click: http://localhost:8080/TIW-servletIntro/SessionCounter?reset=true. At the next request, the counter will restart.
